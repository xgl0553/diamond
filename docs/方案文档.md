# 天天基金基金持仓分析方案文档

## 目标
构建 FastAPI Web 系统，支持登录权限、手工触发采集、正序/倒序排行双页面。

## 已确认规则
- 默认管理员：一次性初始化脚本创建，无注册入口
- 验证码：算术验证码
- 会话策略：Cookie Session
- 手工采集入口：独立页面 `/crawl`
- 触发参数固定：基金范围 all、排序每方向100、持仓前10
- 去重规则：`batch_name + direction` 唯一，重复时提示“数据已采集”
- 上一期定义：同方向下按 `created_at` 倒序选择上一批次

## 页面需求
- 倒序页面：`/rankings/desc`
- 正序页面：`/rankings/asc`
- 每页默认10行，支持10/20/50/100
- 先展示当前页面基金持仓股汇总排行（按持仓市值合计）
- 再展示上一期对比汇总排行（市值变化、持股数变化、变化状态）
- 最后展示基金排行（6字段：1/2/3/4/5/15）

## 数据模型（核心）
- users / roles / permissions / user_roles / role_permissions
- auth_sessions / captcha_challenges
- crawl_batches（batch_name + direction 唯一）
- fund_rank_items（6字段排行）
- fund_holding_items（前10大持仓，含持股数和持仓市值）
